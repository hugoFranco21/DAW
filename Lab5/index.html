Preguntas Lab. 5

  1. ¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?
    Para que no se tengan que hacer llamadas al servidor sabiendo que va a regresar un error, mejor se checa del lado del cliente, y si cumple, ya se manda al servidor para validarse, evitando asi estres innecesario en el servidor. [1]
  2. ¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?
    Una forma es desactivando JavaScript dentro del navegador, otra es usando una proxy para capturar la forma y mandar una con los valores que quieras. [2]
  3. Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?
    Porque es suficientemente bueno para los usuarios que no realizan modificaciones a su navegador, que es la gran mayoria, con lo que acelera bastante el proceso. Y para los usuarios avanzados que saben como saltarse la validacion, esta la validacion del lado del servidor.
