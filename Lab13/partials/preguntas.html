Preguntas

¿Por qué es importante hacer un session_unset() y luego un session_destroy()?

Porque usar únicamente el session_destroy() sólo borra los datos de la sesión, el resto de las variables $_SESSION siguen con el valor que se les da, entonces con otro session_start() se podría recuperar esta información y ser robada maliciosamente. [1]

¿Cuál es la diferencia entre una variable de sesión y una cookie?

La variable de sesión no se guarda en la computadora del usuario, la cookie sí. [2]

¿Qué técnicas se utilizan en sitios como facebook para que el usuario no sobreescriba sus fotos en el sistema de archivos cuando sube una foto con el mismo nombre?

Facebook utiliza un identificador especial para cada archivo, con lo que evita que se repita el nombre de algún archivo. [3]

¿Qué es CSRF y cómo puede prevenirse?

Es un tipo de ataque en el cual el atacante inserta código malicioso en una página donde un usuario esta loggeado para que se ejecute y se haga alguna acción maliciosa, como robar dinero, información, etc. Se puede prevenir usando cookies como verificadores de que sea el usuario el que está realizando los movimientos y usando POST como método de envío de formularios. [4]
